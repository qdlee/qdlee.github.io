文件导航和文件管理

Goto Anthing

Goto Anything让你快速在文件中导航。使用Ctrl + P打开它。当你在输入框中键入的时候，打开状态文件的名称和打开状态目录中的文件会被搜索，最佳匹配的预览会被显示。这个预览是瞬时的，也就是说，直到你在它上面执行一些操作之前，它不会成为实际活动的缓冲区。瞬时视图在你摁Esc的时候会关闭。你会在其他情况下发现瞬时视图，例如当在侧边栏单击一个文件时。

Goto Anything无愧于它的名字-它能做的不只是定位文件。

Goto Anything命令

Goto Anything有一些特别的命令，它们能将你带到文件的其他地方而不仅仅只是文件的开头。任何这种命令都可以与文件检索查询组合使用，并且会应用到当前选择的文件或者如果你没有指定任何文件名搜索条目的话,应用到你当前正在编辑的文件

命令使用特殊的字符调用，例如:，并且该字符后的所有文本都会被命令解释，例子：

island:123

这个指令命令Sublime Text首先查找匹配island的文件然后定位到第123行。

下面是支持命令的列表

@symbol

在活动缓冲区中查找符号；快捷键Ctrl + R

符号通常是类或者函数，但也可以是通过语法定义定义的任何东西。参见Symbols-语法首先项(xxx被添加)。相反地它们完全没有被定义，在这种情况下查找符号会失败。

#search

在文件中匹配search值模糊查找一个单词并高亮所有匹配；快捷键Ctrl + ;

:line_number

转到指定的行，如果超出范围则转到文件末尾；快捷键Ctrl + G

侧边栏

侧边栏提供对你的项目的概览。添加到侧边栏的文件和文件夹在Goto Anything和项目范围内的动作是可用的。项目和侧边栏是紧密相关的。不管是隐式还是显式的，总是有一个打开的项目。

要打开或者关闭侧边栏，按Ctrl+K, Ctrl+B

可以使用箭头在侧边栏内导航，但是首先你需要通过按Ctrl_0给它输入焦点。要将输入焦点返还给缓冲区，按Esc。或者，你可以使用鼠标达到同样的效果，但是为什么你要这么做呢？

侧边栏也通过上下文菜单提供了基本的文本管理操作。

项目

项目把你需要在上面工作的文件和文件夹的集合组合在一起作为一个单元。一旦你以适合自己的方式通过添加文件夹建立了你的项目，保存并给它命名。项目文件使用.sublime-project 扩展名。

使用Project菜单和侧边栏的上下文菜单，你可以对一个项目进行添加和删除文件夹操作。此外，你可以拖拽文件夹到一个窗口之上，它们会自动被添加。

要保存项目，选择Project | Save Project As....

要快速在项目间切换，按Ctrl+Alt+P或者你可以浏览Projects | Recent Projects

通过将.sublime-project 文件作为参数，你可以从命令行打开一个项目。

项目定义

项目定义存储在扩展名为.sublime-project 的JSON文件中。无论在哪儿有一个.sublime-project 文件，你都会发现一个辅助的.sublime-workspace文件，其中包含用户特定数据，比如打开的文件和对每个文件的修改。后者由Sublime Text使用并且你不能编辑。

项目定义支持3个顶级的部分:folders,表示包含的文件夹，setting，表示覆盖的设置，和build_systems，表示项目特有的构建系统。

示例：

			{
			    "folders":
			    [
			        {
			            "path": "src",
			            "folder_exclude_patterns": ["backup"]
			        },
			        {
			            "path": "docs",
			            "name": "Documentation",
			            "file_exclude_patterns": ["*.css"]
			        }
			    ],
			    "settings":
			    {
			        "tab_size": 8
			    },
			    "build_systems":
			    [
			        {
			            "name": "List",
			            "cmd": ["ls"]
			        }
			    ]
			}
Folders

每个文件夹必须有path,视情况可以有folder_exclude_patterns和file_exclude_patterns设置。路径可以相对于项目目录或者是一个绝对路径。文件夹也可以给一个name设置，设置它们怎样在侧边栏显示。

Settings

一个项目可以定义项目特有的设置，只应用到该项目内的（打开的）文件。项目特有的设置覆盖普通的用户设置但不会覆盖语法特有的设置。

你可以覆盖几乎所有设置（除了全局设置）。

Build Systems

你可以在一个项目定义中定义项目特有的构建系统。在普通的构建系统之外，必须为每一个指定name。通过普通的Tools | Build Systems 菜单，列在这的构建系统是可用的。
