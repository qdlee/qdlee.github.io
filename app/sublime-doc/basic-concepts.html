基本概念

在此我们将解释一些概念，读者需要熟悉这些概念以便完全理解本指南的内容。

本指南的约定

本指南是以一个Windows用户的视角来编写的，但是大部分的指令只需要很小的改变就可以在其它平台工作。

除非另外说明，相对路径(e.g.Packages/User)以数据目录开头。

当说明键盘快捷键时，我们假定使用默认键绑定。由于Sublime Text映射键到命令的方式，一些键绑定不会匹配你当地的键盘布局。

功能强大带来许多问题

Sublime Text是一个非常可扩展和可定制的编辑器。你可以立即用它做许多事情，但是如果你花费一些时间来裁剪它以达到你的要求，它会给你超级力量。本指南将教授你所需要的一切来让你知道如何配置Sublime Text.

在下面的段落中，我们将概述你使用Sublime Text一段时间没有意识到的一些方面。不断探索编辑器并且浏览本指南，一切都会在某个时刻逐渐被理解。

Sublime Text当然是程序员的通用工具，但是你不需要成为使用它的一员，或者甚至配置它心使它成为你编写的完美工具。然而，如果你是一名黑客，你会想要花费你一天中的剩余时间来玩这个编辑器。

数据目录

Sublime Text2将几乎所有用户感兴趣的文件存储在数据目录下。下面是平台相关的位置:

  • Windows%APPDATA%\Sublime Text 2
  • OS X~/Library/Application Support/Sublime Text 2
  • Linux~/.config/sublime-text-2

对于便携式安装，查看Sublime Text 2/Data.这里，Sublime Text2部分引用了你将Sublime Text2压缩包解压到的目录。

请注意只有在便携式安装中有一个名为Data的目录存在。对于其它类型的安装，数据目录在上面说明的位置。

包目录

这是一个关键目录:所有支持编程和标记语言的资源都存储在里面。package是一个包含对Sublime Text有特殊意义的相关文件的目录。

你可以从Sublime Text菜单(Preferences | Browse Packages...)来访问包目录，或者使用api调用:sublime.packages_path().在本指南中，在涉及此位置时，我们使用Packages, packages path, packages folder o或者 packages directory。

User包

Packages/User是一个总目录，包含自定义插件、代码片段、宏等等。可以认为它是包文件中的私人区域。Sublime Text在升级的时候，永远不会覆盖Packages/User中的内容。

Python控制台和Python API

这些信息是程序员特别感兴趣的。对于其它的Sublime Text用户，你只需要知道这允许有编程技能的用户向编辑器添加他们自己的特性。(所以去学习怎样编程吧，有趣极了！)

Sublime Text内嵌一个Python解释器。当你编写插件的时候，它是一个检查Sublime Text设置和快速测试API调用的有用的工具。

要打开Python控制台，摁Ctrl+` 或者在菜单中选择View | Show Console

有点困惑？让我们慢慢地再尝试一次：

Python是一种编程语言，以对易上手同时很强大而闻名。API是应用程序接口(Application Programming Interface)的简称，这以一种奇特的方式说明Sublime Text预备由用来来编程。换句话说，Sublime Text通过Python让用户进入它内部。最后，控制台(console)是一个Sublime Text内部的小窗口，让你输入简短的Python代码片断并且运行它们。控制台也通过Sublime Text或者它的插件显示文本输出。

你系统中的Python vs Sublime Text内嵌的Python

在Windows和Linux中，Sublime Text自带Python解释器并且与你系统中的Python安装是分离的。

在OS X中，使用系统中的Python代替。修改你系统中的Python版本，比如使用MacPorts版本替换，可能引起Sublime Text出问题。

嵌入的解释目的只是与插件API交互，不针对一般开发。一些插件的运行会有问题，因为嵌入或者使用的解释器在每个OS中不是都一样。

现在对你来说可能没有意义的包、插件、资源和其它东西

目前，只需要记住几乎Sublime Text中所有东西都可能适应你的需求。这个巨大的灵活性是你为什么会学习如此多的配置文件的原因:配置文件肯定是你指定所有偏好的的地方。

Sublime Text中的配置文件让你改变编辑器的行为，添加宏、代码片断或者创建新特性-特性意味着你能想要的任何东西。好吧，也许不是任何，但是Sublime Text显然给了你大量的控制权。

这些配置文件只是有着特殊结构和格式的文本文件:大部分是JSON，但是你也会发现XML文件。

在本指南中，涉及这些不同的文件时，我们统称为资源。Sumlime Text会在包目录中寻找资源。为了保持整洁，编辑器有一个包(package)的概念，包是一个包含合成整体(也许它们都对更快编写email有帮助或者使用某个编程编程编码)的资源的目录。

Textmate兼容性

这部分信息主要对从Textmate转过来的Sublime Text有用。

Sublime Text能与使用例外命令的Textmate软件集相当地兼容。另外,Sublime Text需要所有语法定义来使用.tmLanguage扩展名，并且所有首选项文件使用.tmPreferences扩展名。这意味着 .plist文件会被忽略，即使它们在Syntaxes或者Preferences subdirectory目录底下。

Vi模拟

这部分信息主要对恐龙和在谈话中使用术语RSI的人们有用。Vi是一个古老的模式编辑器，它允许用户从键盘执行所有操作。Vim，Vi的一个现代版本，仍被广泛使用。

Sublime Text通过Vintage包提供Vi模拟。Vintage包默认被忽略。阅读更多关于Vintage的信息在官方文档中。
