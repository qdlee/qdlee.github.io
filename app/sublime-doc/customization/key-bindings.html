键绑定让你映射按键顺序到动作。

文件格式

键绑定使用JSON格式定义并存储在.sublime-keymap文件中。为了和平台每个更好的集成，对于Linux,OS X和Windows有不同的键映射。只有对应平台的键映射会被加载。

示例

下面是Windows平台默认键映射的摘录

[ { "keys": ["ctrl+shift+n"], "command": "new_window" }, { "keys": ["ctrl+o"], "command": "prompt_open_file" } ]

定义和覆盖键绑定

Sublime Text附带一个默认的键映射(e.g.Packages/Default/Default (Windows).sublime-keymap)。为了覆盖键绑定，在默认的上面定义或者添加一个新的，你可以以较高的优先级将他们存储在一个单独的键映射文件中，例如Packages/User/Default (Windows).sublime-keymap。

查看合并和优先级以获得关于Sublime Text怎样对文件排序以合并的更多信息。

高级键绑定

简单的键绑定由键组合和被执行的命令组成。然而，有一些复杂的语法用来传递参数和提供上下文感知。

传递参数

参数由args关键字指定：

{ "keys": ["shift+enter"], "command": "insert", "args": {"characters": "\n"} }

这里，当你按Shift+Enter时，\n被传递到insert命令。

上下文

上下文决定根据插入符号的位置或者一些其它情况，什么时候一个给定的键绑定是可用的。

{ "keys": ["escape"], "command": "clear_fields", "context": [ { "key": "has_next_field", "operator": "equal", "operand": true } ] }

这个键绑定被翻译成如果有下一个域可用的话就清除代码片断域并且继续正常编辑。因此当你不是循环通过代码片断域时，按Esc不会触发这个键绑定(然而，如果Esc恰巧也绑定了不同的上下文，另外一些事情可能会替代发生-那可能就是Esc)的情况。
